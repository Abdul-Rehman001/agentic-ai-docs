# Start Here

> **Welcome to the Agentic AI Handbook!** If you just landed here from a shared link, read this page before jumping into the modules.

This is not a traditional API reference, nor is it a random collection of tutorials. It is a **modular, sequential curriculum** designed to take you from understanding raw LLMs to building production-grade, multi-agent RAG systems.

---

## 🎯 Who is this for?

This handbook is designed for **software engineers** (particularly those coming from Web, Frontend, or Full-Stack backgrounds like JavaScript/TypeScript) who want to systematically learn **AI Engineering**. 

It assumes you know how to code (functions, loops, APIs), but it **does not assume you know anything about AI**. If you don't know what a vector embedding is, or what Model Context Protocol (MCP) does, you are in the right place.

---

## 🛠️ Prerequisites

All code in this handbook is written in **Python**, as it is the lingua franca of AI. 

However, if you are a JavaScript/TypeScript developer, or haven't written Python in years, **do not panic**. You do not need to take a generic Python course. Instead, start by reading [Module 0: Python Patterns](./00-python-patterns.md). It maps the exact 8 Python patterns used in this handbook directly to their JS/TS equivalents.

---

## 📖 How to Use This Handbook

The most important rule of this handbook is: **Do not skip ahead.**

Every module builds directly on the concepts, and often the exact code, of the previous modules. 

1. **Start at Module 1:** This establishes the absolute basics of LLM generation and tool calling.
2. **Read the Glossary inline:** Every time a new concept is introduced, it is added to the [Glossary](./glossary.md). If you see a term you don't recognize, check the glossary—it acts as a persistent memory bank for the course.
3. **Study the Architecture:** Pay attention to the diagrams (now upgraded to interactive SVGs!). Agentic AI is an architectural challenge, not just a coding one.
4. **End at Module 10:** By the final module (Production Topics), you will understand how to cache, rate limit, and trace the systems you built in the earlier modules.
